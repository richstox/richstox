# Fork Onboarding Checklist (For New Agents)

When you land in a fork of RICHSTOX, follow this checklist:

## Step 1: Read the 4 canonical files (in order)
- [ ] /app/ONBOARDING.md
- [ ] /app/PROJECT_STATE.md
- [ ] /app/DB_SCHEMA.md
- [ ] /app/TASKS.md

## Step 2: Send BOOTSTRAP acknowledgment to Richard
Use this template:

```
## Bootstrap Acknowledgment

1. Confirm you read all 4 files (list paths)
2. Summarize in 3 bullets:
   - What RICHSTOX is
   - The #1 non-negotiable rule
   - Current single focus (from TASKS.md)
3. Confirm you understand approval gate (proposal → approval → implement)
4. Confirm you will NOT create new DBs, ad-hoc queries, or provider metrics
5. State: "Ready for work. Awaiting Richard's next instruction."
```

## Step 3: Wait for Richard's approval
Do NOT proceed with any code changes, audits, or work until Richard approves.
